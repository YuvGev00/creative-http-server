⚒ Streaming Lab

Two real-time HTTP techniques, both hand-built on raw net sockets — no WebSocket here, no libraries.

① Chunked Transfer Theater

The server sends Transfer-Encoding: chunked and writes each piece by hand (size\r\n data \r\n). The page below assembles itself live — open DevTools › Network to see the handcrafted chunks arrive.

② Server-Sent Events

A long-lived text/event-stream response. The browser's native EventSource receives named events the server pushes once per second — one-way real-time over plain HTTP.
